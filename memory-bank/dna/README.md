# DNA Index

DNA is the constitution of project documentation. It defines principles, documentation rules, frontmatter schema, and lifecycle.

- [Principles](principles.md) — foundational principles of the project: SSoT, atomicity, progressive disclosure. Read first.
- [Document Governance](governance.md) — SSoT implementation, dependency tree. Answers the question: who owns which fact.
- [Frontmatter Schema](frontmatter.md) — schema of frontmatter fields.
- [Document Lifecycle](lifecycle.md) — maintenance rules, sync checklist.
- [Cross-references](cross-references.md) — rules for bidirectional navigation between code and docs.
