# Runbooks Index

This directory holds runbooks for repeatable operational tasks.

A runbook must answer these questions:

- What is the trigger?
- What to check first?
- Which commands to run?
- What result to expect?
- How to safely roll back?
- Who to escalate to and when?

## Suggested Structure

1. Summary
2. Trigger / symptoms
3. Safety notes
4. Diagnosis
5. Resolution
6. Rollback
7. Escalation

If the project has no runbooks yet, this directory can contain only this index.
